### 安装

```bash
# 直接安装 (YOLO)
curl -fsSL https://novahub.ai/install | bash

# 软件包管理器
npm i -g novahub@latest        # 也可使用 bun/pnpm/yarn
scoop install novahub             # Windows
choco install novahub             # Windows
brew install anomalyco/tap/novahub # macOS 和 Linux（推荐，始终保持最新）
brew install novahub              # macOS 和 Linux（官方 brew formula，更新频率较低）
paru -S novahub-bin               # Arch Linux
mise use -g novahub               # 任意系统
nix run nixpkgs#novahub           # 或用 github:anomalyco/novahub 获取最新 dev 分支
```

> [!TIP]
> 安装前请先移除 0.1.x 之前的旧版本。

### 桌面应用程序 (BETA)

NovaHub 也提供桌面版应用。可直接从 [发布页 (releases page)](https://github.com/anomalyco/novahub/releases) 或 [novahub.ai/download](https://novahub.ai/download) 下载。

| 平台                  | 下载文件                              |
| --------------------- | ------------------------------------- |
| macOS (Apple Silicon) | `novahub-desktop-darwin-aarch64.dmg` |
| macOS (Intel)         | `novahub-desktop-darwin-x64.dmg`     |
| Windows               | `novahub-desktop-windows-x64.exe`    |
| Linux                 | `.deb`、`.rpm` 或 AppImage            |

```bash
# macOS (Homebrew Cask)
brew install --cask novahub-desktop
# Windows (Scoop)
scoop bucket add extras; scoop install extras/novahub-desktop
```

#### 安装目录

安装脚本按照以下优先级决定安装路径：

1. `$NOVAHUB_INSTALL_DIR` - 自定义安装目录
2. `$XDG_BIN_DIR` - 符合 XDG 基础目录规范的路径
3. `$HOME/bin` - 如果存在或可创建的用户二进制目录
4. `$HOME/.novahub/bin` - 默认备用路径

```bash
# 示例
NOVAHUB_INSTALL_DIR=/usr/local/bin curl -fsSL https://novahub.ai/install | bash
XDG_BIN_DIR=$HOME/.local/bin curl -fsSL https://novahub.ai/install | bash
```

### Agents

NovaHub 内置两种 Agent，可用 `Tab` 键快速切换：

- **build** - 默认模式，具备完整权限，适合开发工作
- **plan** - 只读模式，适合代码分析与探索
  - 默认拒绝修改文件
  - 运行 bash 命令前会询问
  - 便于探索未知代码库或规划改动

另外还包含一个 **general** 子 Agent，用于复杂搜索和多步任务，内部使用，也可在消息中输入 `@general` 调用。

了解更多 [Agents](https://novahub.ai/docs/agents) 相关信息。

### 常见问题 (FAQ)

#### 这和 Claude Code 有什么不同？

功能上很相似，关键差异：

- 100% 开源。
- 不绑定特定提供商。推荐使用 [NovaHub Zen](https://novahub.ai/zen) 的模型，但也可搭配 Claude、OpenAI、Google 甚至本地模型。模型迭代会缩小差异、降低成本，因此保持 provider-agnostic 很重要。
- 内置 LSP 支持。
- 聚焦终端界面 (TUI)。NovaHub 由 Neovim 爱好者和 [terminal.shop](https://terminal.shop) 的创建者打造，会持续探索终端的极限。
- 客户端/服务器架构。可在本机运行，同时用移动设备远程驱动。TUI 只是众多潜在客户端之一。
